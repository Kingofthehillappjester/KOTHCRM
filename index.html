The beating heart — Ops backend & Personal Agent

Two more five-minute deploys turn this from a CRM into the company's operating system: Ops powers shared Projects (notes, tasks, file uploads to Drive) and the Playbook; the Personal Agent pipes each person's own Gmail + Calendar into the Daily Brief.

A

Ops backend — one for the whole team

Create a Google Sheet called KOTH Ops → Extensions → Apps Script → paste the code → Deploy → New deployment → Web App → Execute as: Me → Who has access: Anyone. Optional: set a TOKEN inside the script for extra privacy. One person deploys; you both use the same URL.

B

Personal Agent — one each, from your own Google account

Go to script.google.com → New project → paste the code. Near the top, change var TOKEN = 'CHANGE-ME' to a password you invent — any random string, e.g. rg-koth-7f2k9q. It is not the Deployment ID or Script ID. Then Deploy → New deployment → Web App → Execute as: Me → access: Anyone, and paste the same token below. ⚠ If you change the code or token later, it only goes live via Deploy → Manage deployments → ✎ → Version: New version → Deploy. Matt repeats this from his own account. The URL + token together unlock your inbox headers — treat both like a password.
